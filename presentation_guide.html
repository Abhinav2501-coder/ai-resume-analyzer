AI Resume Analyzer Guide

1. Architecture & Tech Stack

Q: Can you walk us through the tech stack used in this project?
React Router v7 Tailwind CSS Zustand Puter.js Our stack is built for performance and modern developer experience. We use React Router v7 for routing and SSR, Tailwind CSS for styling, and Zustand for lightweight global state management. The backend and AI features are powered by Puter.js, which provides instant authentication, KV storage, and direct access to state-of-the-art vision models.
Q: Why did you choose Zustand instead of Redux or Context API?
Zustand gives us the best of both worlds: the simplicity of Context API without the "provider hell" or unnecessary re-renders, and the power of global state without the overwhelming boilerplate of Redux. It makes our state management (loading states, user auth, errors) extremely clean and scalable.

2. Data Processing & Privacy

Q: How do you handle PDF parsing and keep it private?
Privacy is a core feature. We use pdfjs-dist to process PDFs entirely in the user's browser. We use a Web Worker to ensure that heavy rendering doesn't freeze the UI. The PDF is converted into a high-resolution image locally, and no sensitive file content is ever uploaded to our own servers before processing.
Q: Why convert PDF to Image instead of just extracting text?
Text extraction from resumes is notoriously difficult because of complex layouts (columns, bars, icons). By rendering the PDF to a high-resolution PNG via HTML5 Canvas, we preserve the visual hierarchy. Vision AI models can then analyze the resume exactly as a human recruiter would, reading layout cues that pure text extraction would miss.

3. Artificial Intelligence

Q: Which AI model is used and how is it optimized?
We integrate Vision LLMs through puter.ai.chat. To get the best results, we use advanced Prompt Engineering. We instruct the model to act as a senior technical recruiter and enforce a strict JSON output structure. This allows our frontend to present the data beautifully (gauges, bullet points, scores) rather than just raw text.

4. Future Roadmap

Q: If you had more time, what features would you add?
First, I would add Multi-page PDF support by stitching images together. Second, I would add Job Description Matching, where users can upload a JD and get a percentage match. Finally, I'd implement an AI-driven one-click resume optimizer to rewrite bullet points for the user.
